git --global config user.name="" 进行配置，加上global即是全局配置
git config --list查看配置过的信息
git status 查看暂存区的状态
git add xxx 将文件添加到缓冲区
git add .  # 添加所有文件
git rm --cached xxx 将文件在缓冲区中删除
git cimmit -m [注释]将文件进行提交
git log 查看提交日志
git log --oneline查看提交日志只显示一行
git restore xxx   将误删除的文件进行恢复
git reset --hard [版本号] 将项目恢复到指定版本号，将会丢失版本号后的版本
git revert [版本号] 是一次还原操作，不会丢失历史记录等同于创建了一份新的提交， 还原某一次的操作
git branch -a 查看所有分支（包括远程分支）
git branch [分支名称] 创建一个分支
git branch -d [分支名称] 删除一个分支
git checkout [分支名称]  切换到分支
git commit -a -m "提交说明：描述本次修改的内容"  一键提交
git config --global alias.ac '!git add -A && git commit -m' 设置后可以一键提交
git ac "提交说明" 一键提交的指令
git merge [合并分支名称] 合并分支
git tag [别名] [历史版本] 解释 为某个历史版本 【git log】添加标签，方便使用
git checkout -b [标签名] 使用标签名进行创建分支
git stash 命令允许你临时保存当前工作目录的更改，以便你可以切换到其他分支或处理其他任务
git stash list 查看存储的进度
git stash pop 应用并删除最近一次存储的进度
git stash clear 清空所有存储

git clone <远程仓库URL> 克隆整个远程仓库（这会自动创建一个与远程默认分支对应的本地分支）

git push -u origin 分支名  首次推送可能需要指定分支
git push 后续推送可以简化
git remote add origin 远程仓库URL 如果是第一次设置远程仓库，需要先关联

git push <远程仓库名> <本地分支名>:<远程分支名> 例如git push origin main:main

git remote add custom-remote https://github.com/2596844296/gitTest.git 添加远程仓库
可缩写为git push origin main
git remote查看远程仓库列表
git remote -v查看远程仓库是否可用
